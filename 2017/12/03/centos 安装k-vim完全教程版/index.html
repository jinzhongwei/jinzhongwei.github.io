centos 安装k-vim完全教程版

Blog

k-vim把vim的插件集成一起，安装经常失败，其中最大的原因是就YouCompleteMe安装失败的几率最大，本人经过多次的折腾终于给摸透了，下面给大家一起分享安装中的各个细节。

一、确保你的python版本为大于2.7+，并且是动态编译的

动态编译的很重要，下面给出编译python的过程

1. 下载python2.7+

下载命令wget，python的官网，自己下载即可，https://www.python.org/

2. 解压文件

以python2.7.5为例，tar -zxvf Python-2.7.5.tar.bz2

3. 创建安装目录

mkdir /usr/local/python27

4. 安装python

./configure –prefix=/usr/local/python27 –enable-shared make make install 其中 –enable-shared会使得对python进行动态编译，生成 libpython2.7.so.1.0 和 libpython2.7.so 文件。

5. 修改老版本的ln指向（注意：这里修改后，可能会影响yum的使用）

ln -s /usr/local/python27/bin/python /usr/bin/python

6.注意：很多人在动态编译完之后，发现还是不行，安装YCM时，继续报同样的错误，那是应为你没有把动态编译的库放在正确的位置。

6.1. cp /usr/local/python27/lib/libpython2.7.so.1.0　 /usr/local/lib

 cd /usr/local/lib  

 ln -s libpython2.7.so.1.0 　libpython2.7.so

6.2. 使用命令whereis libpython2.7.so.1.0得到如下结果就说明 libpython2.7.so.1: /usr/local/lib/libpython2.7.so.1.0

6.3. 如果whereis没有结果，或者还有import错误，可以尝试如下操作： 在/etc/ld.so.conf中加入新行/usr/local/lib

保存后，运行  

/sbin/ldconfig  

/sbin/ldconfig –v

二、确保你的vim版本够高，最好还是直接安装vim8.0

gitclone git@github.com:jinzhongwei/vim.git cd vim/src ./configure –with-features=huge –enable-rubyinterp –enable-pythoninterp –enable-luainterp –enable-multibyte –enable-sniff –enable-fontset –enable-cscope –enable-gui=auto –prefix=/usr –enable-xsmp –enable-xim –enable-gpm

make

make install

三、安装k-vim

下载

git clone git@github.com:jinzhongwei/k-vim.git

安装依赖

sudo yum install python-devel.x86_64

sudo yum groupinstall ‘Development Tools’

sudo rpm -Uvh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

sudo yum install the_silver_searcher

sudo yum install cmake

pip install flake8 yapf

安装

cd k-vim/

sh -x install.sh

安装完之后，运行命令vim，如果没有报什么问题，恭喜你安装成功。 如果出现什么问题，根据问题进行解决，注意很重要的一点要对YCM重新编译一遍， cd k-vim/bundle/YouCompleteMe/

./install.sh –clang-completer

四、注意

在安装之前还需要注意自己的电脑的CMake版本是不是过低，以及GCC版本过低，如果过低，请自行升级。 我自测的GCC版本4.8.5可以，CMake3.0可以
